02 — Track A
SOUL抽出方法の実験
5段階の実験を順次実施し、SOULフォーマットの実現性と価値を検証した。
A1 — 最小パイプライン 完了
テキスト入力 → Ollama抽出 → ベイズ更新 → soul.json保存のループを実装。ユニットテスト38件全通過。4層構造のSOULスキーマ（Core Identity / Episodic Memory / Semantic Map / Temporal Patterns）をJSON形式で定義。
A2 — 抽出精度検証 完了
10種類のテキスト（内向的〜冒険的、日常〜人生の転機）で自動評価。感情カテゴリ 100%、重要度レンジ 100%、性格方向 75%。総合精度 92%。
A3 — 多元入力統合 完了
テキストのみ vs テキスト+位置+時刻のコンテキスト付きを比較。性格シグナルは全5ケースで変化（100%）、重要度は3/5で変化。「おいしいご飯」+「ミシュラン三つ星」でopenness 0.20→0.70。多元データの有効性を実証。
A4 — ベイズ的忘却 完了
全13テスト通過。20回の一貫した観測でσが0.30→0.055に収束（確信が6倍に向上）。矛盾する観測への適応、exponential decay、distillation（エピソードが性格に溶け込む）すべて設計通り動作。
A5 — Claude連携体験検証 完了
SOULから自動生成されたSystem PromptでAIの応答が明確にパーソナライズされることを確認。SOULなしの汎用応答と比較して、興味・価値観・行動パターンが応答に反映された。
