================================================================================
KEY CONCEPTS AT A GLANCE
================================================================================

8 CATEGORIES:
  core_identity, episodic_memory, emotional_state, behavioral_patterns,
  health_vitals, location_movement, social_graph, interests_values

4 GRANULARITY LEVELS:
  Full → raw data
  Summary → aggregated/averaged
  Anonymized → PII removed, hashed
  Hidden → non-disclosure

5 SCOPE TEMPLATES:
  Claude Personal (trusted partner, no location/health)
  Work Assistant (work context only)
  Health Coach (vitals + emotion, no social)
  Minimal (core identity summary only)
  Emergency (full data, 1-hour auto-expiry)

CAPABILITY TOKEN:
  - HMAC-SHA256 signed permission document
  - JWT-compatible format
  - Revocable, time-limited, one-time option
  - Single source of truth for access control

PERMISSION GATEWAY:
  - Central access control (called on every SOUL access)
  - Token verification → scope extraction → data filtering
  - Applies granularity transformations
  - Returns filtered SOUL with only permitted categories


================================================================================
BOUNDARY CASE SUMMARY
================================================================================

7 documented cases where data spans multiple categories:

1. "仕事の会話から推測される健康状態" → HEALTH_VITALS rules apply
2. "Aさんとの会話" → SOCIAL_GRAPH rules apply (person name/ID involved)
3. "場所から推測される行動パターン" → LOCATION_MOVEMENT rules apply
4. "睡眠ログ（感情 + 健康）" → HEALTH_VITALS rules apply
5. "休日に一人でいる傾向" → SOCIAL_GRAPH rules apply
6. "Spotifyプレイリスト（友人と共有）" → SOCIAL_GRAPH rules apply
7. "推測される精神状態（複数弱信号）" → HEALTH_VITALS + EMOTIONAL_STATE apply

DECISION RULE: Apply the most restrictive category's permissions
