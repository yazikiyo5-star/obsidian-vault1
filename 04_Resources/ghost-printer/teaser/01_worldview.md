# Ghost-Print / Worldview & Tone

> 方向性: **サイバーパンク・ダーク**
> 系譜: *Ghost in the Shell (1995)* / *Blade Runner 2049* / *Serial Experiments Lain* / *Cyberpunk: Edgerunners*
> 言語: 日英バイリンガル（一次は日本語、英語はサブコピー）

---

## 1. ポジショニング・ステートメント

> **AI に魂を奪われる時代に、自分の魂を「印字」する装置。**

ディストピアではない。ディストピアの中で、ユーザーが**主導権を取り戻す**装置。

- 既存の AI: あなたから無断でデータを抜く（受動）
- Ghost-Print: あなたが**抽出し、選び、渡す**（能動）

サイバーパンクの主人公は常に「企業に身体を売られた個体」だった。Ghost-Print のユーザーは **自分の身体（リング）と意識（音声）を、自分のリッパードックで印字する個体**である。これは "Self-Ripperdoc" のメタファー。

| **B** | **What will you give them?** | **主タグライン候補・英語** | 問いかけ |

**主タグラインの考え方:**
A「人類、 あなたの選択は？」 は エヴァ的 ＝ 視聴者を 「観察される側」 ではなく 「決断する側」 に置く。 サイバーパンクの主人公は常に選択を迫られる存在だった。 Ghost-Print のユーザーも、 AI の前に立たされて 「渡すか、 戻すか」 を毎回問われる。 タグラインがその構造そのもの。

副コピー（マニフェスト断章）:
- "毎日、 AI はあなたから何かを抜き出している。"
- "Bonsai が剪定する。 あなたが選ぶ。 AI が読む。"
- "Ghost Dock を挿す瞬間、 所有権が反転する。"
- "渡すか、 戻すか。 ―― それを毎回選べることが、 唯一の自由だ。"
- "コアは閉じる。 シェルは、 あなたが決める。"

ブランドステートメント (フッター・名刺裏など):
> Ghost-Print is a self-ripperdoc.
> You print. You choose. You give — or you keep.

## 3. ビジュアルランゲージ

### 3.1 配色（コアパレット）

```
─────────────────────────────────────────────
  --void          #050507    背景・黒（純黒は使わない）
  --ink           #0B0D12    パネル
  --steel         #1A1F2B    枠線・サブパネル
  --bone          #E6E8EE    本文白
  --ghost         #8A93A6    補助テキスト
─────────────────────────────────────────────
  --cyan          #00FFE5    アクセント・データ可視化
  --neon-magenta  #FF1F8F    警告・GHOST シール
  --amber         #FFAE00    CRT・端末カーソル
  --blood         #FF3344    エラー・破棄処理
─────────────────────────────────────────────
```

ルール:
- ベースは **#050507 (void)** 一色。Pure black (#000) は禁止（CRT グロー再現のため微妙に青寄りの黒を使う）
- アクセントは **cyan #00FFE5 を主軸 / magenta #FF1F8F は GHOST ファイル関連にだけ使う**
- amber は端末カーソル・ターミナル UI に限定（多用すると 80s アーケード感が出すぎる）
- blood は破棄・拒否・"AI に渡さない" の意思表示にのみ

### 3.2 タイポグラフィ

```
ディスプレイ（見出し）
  英: Berkeley Mono / NB International Mono / IBM Plex Mono
  和: Noto Sans JP（中字〜太字）

本文
  英: IBM Plex Mono 500
  和: Noto Sans JP 400 / Source Han Sans JP 400

データ・ASCII
  Berkeley Mono / JetBrains Mono Light
  half-width katakana (ｶﾀｶﾅ) を意図的に混ぜる
```

タイポルール:
- **見出しはモノスペースのみ**。プロポーショナル見出しは Ghost-Print の世界観に合わない
- 句読点は **半角カンマ・ピリオド** ではなく **全角「、。」と半角 `,` `.` を混在**させる（攻殻的タイポ）
- 数値は必ず **3桁ゼロパディング**: `BPM 078 / SpO2 097`
- 英単語の前後にスペースを **空けない**（半角カナと混在する詰め組）

### 3.3 モチーフ・絵作り

撮るべきもの:
- 指輪 (COLMI R02) を装着した手の **マクロ**（指紋・産毛が見えるくらい）
- 夜の机。モニタの光が顔の半分だけ照らす
- Orange Pi 3B 基板の銅パターン・ヒートシンク・ファン
- Ghost Dock を **挿す瞬間** のスローモーション（金属音）
- GHOST バイナリの **hex dump** が画面を流れる
- Bonsai 1.7B の "盆栽" イメージ — 低ポリゴンの盆栽の枝に光が走り、重要度の高い枝だけが赤く灯る

撮らない/避けるもの:
- 人物の顔全体・笑顔・ライフスタイルカット（Apple ルックに堕ちる）
- Tokyo の俯瞰ネオン（ブレラン直引用は陳腐）
- AI に対する "怖い表情"（演技で見せず、データの抜かれ方で見せる）

### 3.4 ハードウェア哲学 ・ Ring / Core / Shell

Ghost-Print の物理装置は **3 つで 1 セット**。 それぞれの役割と動詞:

| 部位 | 動詞 (en/ja) | 場所 | 役割 |
|------|------------|------|------|
| **RING** | FEEL / 感じる | 肌の上 (24h 装着) | 心拍 ・ SpO2 ・ 体温 ・ 動きを取る。 計算しない |
| **CORE** | EDIT / 編集する | 机の上 (常時起動) | Bonsai 1.7B が剪定 → GHOST.bin 生成。 オフライン |
| **SHELL** | DRESS / 装う | 手の中 / 部屋 | コアを包む外装。 STL 公開、 ユーザーが印刷・改造 |

3 つで **身体 → 認知 → 表現** の連鎖を作る。 この三層は Three Pillars (EXTRACT/DISTILL/CHOOSE = データ動詞) と直交する **物理レイヤー** で、 ティザー・LP・ドキュメントすべてで両方を見せる。

#### CORE と SHELL の関係 ・ 2 層構造

Ghost-Print は **2 層構造** で語る:

```
        ┌──────────────────────────────┐
        │           SHELL              │   ← あなたが選ぶ
        │     ┌──────────────┐         │     （3D プリント / 配布データ）
        │     │     CORE     │         │   ← 我々が閉じる
        │     │  rev.A6 PCB  │         │     （計算・センサ・ストレージ）
        │     └──────────────┘         │
        └──────────────────────────────┘
```

- **CORE** ・ Ghost-Print rev.A6 のメイン基板 + センサ + ストレージ。 我々が責任を持って閉じる
- **SHELL** ・ コアを包むケース。 **STL データを公開**、 ユーザーが好きに 3D プリントできる。 木目・金属切削・透明・コスプレ用 ・ 何でもいい

これは Light Phone の "禁欲的に閉じる" 哲学とも、 Framework Laptop の "全部開ける" 哲学とも違う。
**機能は閉じる、 表現は開ける** ―― ちょうど SOUL 哲学 (= GHOST 哲学) の物理版。

シェル配布チャンネルは **`/shells`** ページ ＝ Thingiverse 的にコミュニティが作ったケースが並ぶ。 公式シェルは 3-4 種を初期提供。

サイバーパンク文脈での意味:
- 義体 / アサルトスーツ / 着せ替え ＝ コミュニティが繰り返してきたモチーフ
- "あなたのゴーストが、 あなたのシェルに住む"
- 拡散・改造を歓迎する姿勢が、 ハッカー/メイカー層への信頼の証になる

### 3.5 グラフィック処理

- **CRT スキャンライン**: 全カットに薄く（opacity 4-8%）
- **走査ノイズ**: 端末ウィンドウだけ、低ビットレート風の走査線歪み
- **Hex dump オーバーレイ**: 画面端に常時うっすらと GHOST ファイルの hex が流れる
- **チャート/ウォッチポイント可視化**: 心拍・SpO2・音声 VAD を CRT 風グラフで（A7 ウォッチポイント設計と整合）
- **Half-width katakana ノイズ**: 画面端にちらつく `ｺﾞｰｽﾄ ｲﾝｼﾞ` `ﾀﾏｼｲ ﾆ ｱｸｾｽ` のような断片

## 5. ナラティブ・ビート（脚本の前段）

ティザーは以下の **5 つのビート** で構成:

```
[0] HOOK     誰かが、毎日、あなたから何かを抜いている
[1] SYMPTOM  心拍・声・履歴が外へ流れ出る描写
[2] DEVICE   Ghost-Print 登場。Bonsai が剪定する
[3] AGENCY   Ghost Dock を挿し、GHOST を取り出す
[4] CHOICE   渡すか、ポケットに戻すか。あなたが決める
```

このビートを 30-45 秒に圧縮するのが脚本タスク（02_script.md）。

## 6. ブランド・ボイス

書き言葉のルール:
- **断定する**。「〜かもしれない」を使わない
- **問いを立てる**。「なぜあなたの魂は、あなたのものではないのか？」
- **数字で語る**。「Bonsai 1.7B、tg 3.03 t/s、72℃」のように生スペックを文中に置く
- **謝らない**。Pre-MVP であることを「未完成」ではなく「進行中」と書く
- **絵文字禁止**。技術 ASCII（`├──`, `▓▒░`, `◉○`）は OK

避けるトーン:
- スタートアップ的「世界を変える」
- TED 的「imagine a world where」
- 仏教的・癒し系（盆栽は使うが "癒し" の文脈に置かない）

## 7. 制約と倫理メッセージング

ティザーとサイトで **明示的に伝えるべき** 約束:
1. **オフライン抽出**: Bonsai 1.7B はデバイス内動作。クラウドに送らない
2. **選択的開示**: 何を渡すかはユーザーが決める（C トラック）
3. **オープン**: GHOST フォーマットは公開仕様（v0.1）
4. **Pre-MVP**: いま動いているものと、これから作るものを区別する

これらは "fine print" ではなく **本文に組み込む**。サイバーパンクの皮で、約束は誠実に。
